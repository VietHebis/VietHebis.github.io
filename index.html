Bootstraps-Beginner

Nếu bạn là một nhân viên liên quan đến IT ít nhiều sẽ nghe qua từ Bootstrap. Vậy thật sự nó là gì hôm nay mình sẽ giới thiệu cho các bạn làm quan và bước cài đặt.

1. Bootstrap là gì ?

  • Bootstrap là một thư viện JS , CSS giúp cho việc phát triển giao diện một cách nhanh chóng và dễ dàng
  • Điểm mạnh của bootstrap là reponsive gird system, nghĩa là trang web của bạn sẽ đảm bào hiển thị tốt trên đủ kích cỡ màn hình từ bé như smartphone vài inch đến TV vài chục inch.
  • Có thể sử dụng bootstrap nhanh chóng bằng CDN của Google, Bootstraps, v.vvv hoặc chúng ta có thể tải source về để trên website của mình thôm qua npm, gem, hoặc tải từ Github về.
  • More info: Bootstrap

2. Cách cài đặt Bootstrap:

2.1 Sử dụng CDN:

  • Các bạn tạo file test.html vào root folder

  • Các bạn lưu ý khi sử dụng thì CDN phải nằm trong tag head của html:

  • Mở browser và kiểm tra nếu như hình thì đã cài đặt thành công. Good luck !

Docker

Beginner Docker

Chào các bạn hôm nay mình sẽ giới thiệu về Docker, một trong những công cụ tuyệt vời, cần thiết cho developer và sysadmin.

1 Docker là gì ?

  • Docker dễ hiểu thì như một máy ảo nhưng ở tầng ứng dụng không như VMWare hay VirtualBox ảo hóa ở machine.
  • Chúng ta có thể tự triển khai ở máy cá nhân, server hoặc cloud.
  • More info: Docker page

2 Chức năng, vai trò của Docker :

  • Cho phép phát triển, di chuyển và chạy các ứng dụng dựa vào công nghệ ảo hóa container trong Linux.

  • Tự động triển khai các ứng dụng bên trong các container bằng cách cung cấp thêm một lớp trừu tượng và tự động hóa việc ảo hóa “mức hệ điều hành”.

  • Docker có thể sử dụng được trên cả 3 hệ điều hành phổ biến: Windows, Linux và Mac OS.

  • Lợi ích của Docker:

    + Nhanh trong việc triển khai, di chuyển, khởi động container
+ Bảo mật
+ Lightweight (tiết kiệm disk & CPU)
+ Mã nguồn mở
+ Hỗ trợ APIs để giao tiếp với container
+ Phù hợp trong môi trường làm việc đòi hòi phải liên tục tích hợp và triển khai các dịch vụ, phát triển cục bộ, các ứng dụng multi-tier.

    3 Các khái niệm cần biết khi sử dụng Docker :

    Image

    • Image trong Docker còn gọi là Mirro. Nó được hiểu như một template dùng để tạo ra container.
    • Được xây dựng từ một loạt các layers. Mỗi layer là một kết quả đại diện cho một lệnh trong Dockerfile.
    • Lưu trữ dưới dạng read-only template.

    Registry

    • Docker Registry là nơi lưu trữ các image với hai chế độ là private và public.
    • Là nơi cho phép chia sẻ các image template để sử dụng trong quá trình làm việc với Docker.
    • Registry phổ biến nhất : Docker Hub
